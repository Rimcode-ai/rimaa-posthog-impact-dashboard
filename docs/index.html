        Guardrails

        • Scope guard — rejects questions outside the dataset.
        • Prompt-injection guard — strips system-override patterns.
        • Slice containment — the Analyst only sees the Researcher's filtered slice, never the full dataset, so it can't reference engineers outside scope.
        • System-prompt grounding — both LLM agents are instructed to only mention logins that appear in their inputs, with concrete numbers.
        • LangSmith tracing (optional) — full agent trace if you supply a key.

        Methodology

        Open and reproducible. The repo contains the fetcher, analyzer, and rendered data.json.

        Window & eligibility

        Merged PRs in the last 90 days. Engineers with fewer than 3 merged PRs are excluded (signal too noisy). Bots and the posthog service account are excluded.

        The 5 signals (each z-scored, then weighted)

        • Surviving code (25%) — additions per merged PR, capped at 2,000 lines/PR. PRs whose title is reverted by a later "Revert ..." PR receive 40% credit.
        • Review leverage (25%) — PRs where the engineer left an approving review with at least one comment ≥25 chars (filters out "lgtm", "👍").
        • Cross-area reach (15%) — distinct top-level product directories touched. Config / lockfiles are filtered out.
        • Incident work (20%) — merged PRs that were themselves, or closed an issue, labeled bug / incident / p0 / p1 / sev / hotfix / regression / outage.
        • Review-graph centrality (15%) — PageRank (α=0.85) on the directed reviewer→author graph.

        Composite

        Σ weight × z-score, then min-max normalized to 0–100 for display. Each card's stacked bar shows positive weighted contributions; below-pool-mean signals are clipped from the bar but reflected in the score.

        Ask AI agent & guardrails

        • Pipeline: deterministic Pre-guard → Researcher (LLM, JSON-strict) → Analyst (LLM).
        • Pre-guard: regex scope check (must reference engineers, impact, posthog, or other dataset terms) + prompt-injection token strip.
        • Slice containment: the Researcher selects logins from a roster of dataset engineers and emits strict JSON. The Analyst then only ever sees that filtered slice — it cannot reference engineers outside it because it never sees them.
        • System-prompt grounding: both agents are instructed to only mention logins from their inputs and to cite specific numbers from the data.
        • LangSmith tracing via direct REST POST when the user supplies a key; provider + model attached to every run.

        Limitations

        • Code semantics are not parsed; "complexity" is approximated by reach + the per-PR cap.
        • Outside contributors and very new hires are under-counted; the <3 PR floor excludes them.
        • Review centrality is sensitive to who reviews whom; a tight clique can reinforce each other.
        • Window is 90 days — engineers on parental leave / sabbatical / cross-org work are not visible here.
