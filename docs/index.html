Window and eligibility

  • 90-day trailing window into PostHog/posthog default branch.
  • Eligibility: 3+ merged PRs in window.
  • Bots, github-actions, and the posthog service account login excluded.
  • Lockfile-only PRs excluded from cross-area math.

The 5 signals (each z-scored, then weighted)

  • S1 Surviving code (25%): per-PR additions capped at 2000 lines. PRs reverted by a later "Revert ..." or "revert:" PR within window contribute at 40% credit, not zero.
  • S2 Review leverage (25%): count of distinct PRs by other authors in window where the engineer left an APPROVED review with at least one body of 25+ chars. Filters out lgtm, emoji-only, single-word rubber stamps.
  • S3 Cross-area reach (15%): distinct top-level repo directories touched. Lockfile-only and config-only PRs do not contribute.
  • S4 Incident work (20%): merged PRs labeled (or that closed an issue labeled) bug, incident, p0, p1, sev, hotfix, regression, outage. Case-insensitive.
  • S5 Review-graph centrality (15%): PageRank with alpha 0.85 on the directed reviewer-to-author graph from substantive reviews (same filter as S2).

Composite

  • raw(e) = sum_i w_i * z(S_i(e))
  • display(e) = round(min-max-normalise(raw, 0, 100))

Ask AI agent and guardrails

  • Stage 1, Pre-guard (no LLM call): scope regex match plus injection strip.
  • Stage 2, Researcher (LLM, JSON-strict): roster of logins only, output { logins, reason }, slice containment forces every login back into the dataset roster, top-5 fallback when empty.
  • Stage 3, Analyst (LLM, prose): inputs are sanitized question plus only the filtered slice from Stage 2. Prose grounded in slice fields.
  • Slice containment is the load-bearing guarantee: the Analyst can only mention logins that exist in the Researcher's slice, because it never sees the others.
  • Token / cost guard: max_tokens 600; slice truncated to 8 KB before sending.
  • Keys live in sessionStorage only and are never logged.

Limitations

  • Code semantics not parsed.
  • Outside contributors under-counted.
  • Review centrality sensitive to cliques.
  • 90-day window excludes leave or sabbatical.
